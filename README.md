# Sales & Customer Analysis using SQL

## About This Project
I built this project to practice SQL in a more practical way. Instead of only focusing on syntax, I tried to understand what the data is saying and how it can support business decisions.

SQL project analyzing sales and customer behavior with business insights

## Dataset
The project uses three tables:
- Customers
- Orders
- Order Items

## What I Explored
Using SQL, I worked on questions such as:
- Total revenue generated
- Top customers by spending (CLV)
- Most sold product by quantity
- Revenue by category
- City-wise revenue performance
- Monthly sales trend
- Identification of repeat customers and churn risk
- Top 20% customers (Pareto analysis)
- Contribution percentage by category

## Key Insights
- A small group of customers contributes a large share of revenue
- Electronics generate higher revenue compared to other categories
- Some customers place only one order, indicating potential churn
- Certain cities contribute more to revenue than others
- Some products sell in higher quantity but generate lower revenue

## Tools and Concepts Used
- SQL (MySQL)
- Joins (INNER JOIN)
- Aggregations (SUM, COUNT)
- GROUP BY and HAVING
- Window functions (RANK, NTILE, running total)
- Date functions for time-based analysis

## Project Structure
- schema.sql: table creation
- data.sql: data insertion
- analysis.sql: SQL queries used for analysis

## What I Learned
This project helped me move from writing queries to thinking in terms of business problems. It improved my understanding of how to:
- Structure data analysis
- Ask the right questions
- Translate results into insights
